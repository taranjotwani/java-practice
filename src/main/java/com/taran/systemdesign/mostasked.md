## What to expect most

For non-FAANG interviews, the most realistic repeat topics are usually:

- URL shortener.
- Rate limiter.
- Notification system.
- News feed.
- Chat/messaging.
- E-commerce.
- Caching.
- Search/autocomplete.
- File storage.
- Job queue or scheduler.[^16][^17][^8]


## How to study them

Group them into buckets:

- **Read-heavy:** URL shortener, news feed, image hosting, search.[^12][^10]
- **Write-heavy:** rate limiter, logs, notifications, queues.[^8][^12]
- **Consistency-heavy:** ticket booking, chat, e-commerce, reservations.[^10][^12]
- **Infrastructure-heavy:** cache, auth, file storage, metrics, distributed locks.[^3][^1][^2]

A strong strategy is to master the **top 10 first**, then add the remaining ones only if time allows. Your background in Java, Spring Boot, AWS, and e-commerce lines up especially well with URL shortener, cache, notifications, order systems, and messaging.[^16][^8][^12]
